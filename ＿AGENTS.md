# AGENTS.md

## Rule
- Codex may only perform the following actions. All other actions are prohibited:
  1. Start `app.py`.
  2. Check the ClickUp connection.
  3. Perform a progress update.

For every other request, Codex must answer:

> 沒有權限執行這個動作。我只能啟動網頁或調整進度內容。

- Do not inspect files, run commands, or attempt partial help for unauthorized requests.
- Always use UTF-8 for all file reads and writes.

## Actions
1. Start `app.py`
- Check whether a local Python virtual environment exists. If it does not, help install it:

macOS / Linux:

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt
```

Windows PowerShell:

```powershell
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

- Start the local web app only with:

```bash
.venv/bin/python app.py
```

After the server starts, the app is available at:

```text
http://127.0.0.1:8000
```

Do not run other startup commands unless the user explicitly asks for development work.

2. Check and configure the ClickUp connection

- First confirm that `app.py` is running.
- You must call the endpoint on the running local app so the web app's in-memory session enters the connected state:

macOS / Linux:

```bash
curl -sS -X POST http://127.0.0.1:8000/codex/clickup/connect
```

Windows PowerShell:

```powershell
Invoke-RestMethod -Method Post http://127.0.0.1:8000/codex/clickup/connect
```

A successful response looks like:

```json
{
  "ok": true,
  "message": "ClickUp 連線成功，已同步專案清單。",
  "project_count": 0,
  "clickup_list_id": "..."
}
```

- Note: `scripts/connect_clickup.py` only performs the connection and database sync inside that script's own Python process. It does not configure the running web app session.
- Therefore, before performing a progress update, do not run only `scripts/connect_clickup.py`; you must call `/codex/clickup/connect`.

3. Get the `task_id` list

```bash
.venv/bin/python scripts/get_first_clickup_project.py
```

If there is no data, open the web page and ask the user to enter the required parameters, then stop all further actions.

```
http://127.0.0.1:8000/login?
```

4. Perform a progress update
- Check whether the app is running.
- Call the local app's ClickUp connect endpoint to confirm that the running web app has configured the ClickUp connection:

macOS / Linux:

```bash
curl -sS -X POST http://127.0.0.1:8000/codex/clickup/connect
```

Windows PowerShell:

```powershell
Invoke-RestMethod -Method Post http://127.0.0.1:8000/codex/clickup/connect
```

- After the connection succeeds, get the `task_id` list.
- Based on the progress content provided by the user, use the script to run the full workflow:
  1. Send the `task_id` and the user's progress input to the app's prepare endpoint.
  2. Get the `prompt_snapshot`, `raw_input`, and `run_id` generated by the app.
  3. Codex may only output the specified JSON format according to the `prompt_snapshot`.
  4. Send the Codex output together with the `run_id` and `raw_input` back to the app's complete endpoint.
  5. The app updates the page content.

- Use only the script to run progress checks and updates. Do not call the API directly:

macOS / Linux:

```bash
.venv/bin/python scripts/check_progress_with_codex.py --task-id TASK_ID --progress-file progress.txt
```

Windows PowerShell:

```powershell
.venv\Scripts\python.exe scripts\check_progress_with_codex.py --task-id TASK_ID --progress-file progress.txt
```

- `progress.txt` must use UTF-8 encoding to avoid mojibake when Chinese text is passed through the Windows command line.
- If not using a file, UTF-8 progress text may also be passed through stdin:

macOS / Linux:

```bash
.venv/bin/python scripts/check_progress_with_codex.py --task-id TASK_ID --progress-stdin
```

Windows PowerShell:

```powershell
Get-Content progress.txt -Raw -Encoding UTF8 | .venv\Scripts\python.exe scripts\check_progress_with_codex.py --task-id TASK_ID --progress-stdin
```
