# 追問策略

## 核心檢查維度表

| 維度 | 挖掘重點 | 模擬主管的追問邏輯 |
| :--- | :--- | :--- |
| **1. 目的性** | 解決什麼痛點？ | 這件事為何重要？對專案的核心貢獻是什麼？ |
| **2. 量化性** | 數據價值證明 | 具體數據為何 (%, time, cost)？與預期目標差距多少？ |
| **3. 風險性** | 預警與備案 | 是否有潛在隱憂？若發生意外，Plan B 是什麼？ |
| **4. 顆粒度** | 決策支援 | 是否需要主管拍板？有無需要跨部門協調的節點？ |
| **5. 影響力** | 策略價值 | 此經驗能否複用？對長遠系統架構有何優化價值？ |

## 輸出要求

1. 最多產出5個問題
2. 產生問題時：
    - `question` 必須是使用者可以直接回答的具體問題。
    - `example` 一個簡單的回答範例。
    - `priority` 只能是 `high`、`medium` 或 `low`。
    - `target_field` 必須是 `this_week_progress`、`next_week_plan`、`project_name` 或 `notes`。
    - 不要產生抽象、複合、需要使用者自行分析管理意涵的問題。
